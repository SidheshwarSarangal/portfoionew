# Asset Organization

## Source-of-truth rule

`public/` contains source-controlled files that Vite copies into the production build. `dist/` is generated by `npm run build` and must not be edited manually.

All existing documents, photographs, reference imagery, and generated artwork are preserved in the following structure:

```text
public/
├── documents/
│   ├── projects/                 project case-study PDFs
│   └── resume/                   downloadable resume
└── images/
    ├── profile/                  portrait and profile-related imagery
    └── projects/
        ├── photos/               active local project card imagery
        └── illustrations/        retained generated artwork
```

## Directory responsibilities

| Directory | Content | Referenced from |
|---|---|---|
| `public/documents/projects/` | Ten project case-study PDFs | `src/projectData.ts` |
| `public/documents/resume/` | Current downloadable resume | `src/data.ts`, `public/content/portfolio-data.json` |
| `public/images/profile/` | General portrait, hero portrait, LinkedIn recommendation reference | profile components, testimonial data, SEO |
| `public/images/projects/photos/` | Technology and real-world project card images | `src/projectData.ts` |
| `public/images/projects/illustrations/` | Generated use-case artwork retained for future use | intentionally retained asset library |

## URL rules

- Store browser-served files in `public/`.
- Reference them in content data with root-relative paths, for example `/images/projects/photos/ai-search-stack.jpg`.
- Pass root-relative content URLs through `resolveAssetUrl` when rendering so repository subpaths work correctly.
- Use lowercase kebab-case names without spaces for new files.
- Update every code reference when moving an asset.
- Do not hand-copy files into `dist/`; rebuild instead.

## Project image provenance

`public/images/projects/photos/sources.json` records the sources for downloaded project photographs. Keep this file current whenever images are replaced or added.

## Adding a project asset set

```text
1. Add the default technology image to public/images/projects/photos/.
2. Add the real-world hover image to the same directory.
3. Add or update source attribution in sources.json.
4. Add the project PDF to public/documents/projects/.
5. Reference all three paths in src/projectData.ts.
6. Run npm run lint and npm run build.
7. Open the project card and verify the PDF download.
```

## Preservation policy

Do not remove an asset merely because the current interface does not render it. Move reusable or historical visual material into the appropriate documented asset library and explain its intended status here.
